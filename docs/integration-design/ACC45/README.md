# ACC45 Integration Design Checklist

This folder contains the generator script for the ACC Interface – ACC45 Claims Integration Design Information Checklist.

## Files

| File | Description |
|------|-------------|
| `generate_acc45_checklist.js` | Node.js script that generates the `.docx` checklist document |

## Usage

### Prerequisites

```bash
npm install -g docx
```

### Generate the document

```bash
node generate_acc45_checklist.js
```

Outputs `ACC45_Integration_Design_Checklist.docx` in the current directory.

## Document Structure

The generated document covers all 13 sections of the Integration Design Information Checklist template:

| Section | Title |
|---------|-------|
| 1 | Workflow Overview |
| 2 | Trigger Events (Profile Behaviour) |
| 3 | Message Creation & Submission |
| 4 | Interaction Pattern |
| 5 | Data Requirements |
| 6 | Response & Status Handling |
| 7 | Amendment / Resubmission Behaviour |
| 8 | Error Handling & Retry Behaviour |
| 9 | Integration Mechanism |
| 10 | Environment & Configuration |
| 11 | Security & Authentication |
| 12 | Operational Behaviour |
| 13 | Reference Materials |

## Notes

- The script in this repo is an **abbreviated reference version** (Sections 1–2 fully included; Sections 3–13 summarised).
- The **full 13-section script** was generated in a Claude session (April 2026) and can be regenerated by asking Claude to reproduce it using the `profile-integration-design` skill.
- Source template: `Integration_Design_Checklist.docx` (uploaded April 2026).
- Profile objects referenced: `ISApproval`, `ISApprovalsFilter` (AliasCode, Reference, PatientId, Comment, EventDate, Diagnosis1/2/3, ShortCodeFormCode).
